Average execution time per difficulty level

Time is of the essence! This chart shows how much time a solution takes (in milliseconds) per difficulty level.
Harder difficulty problems require more time to complete because programmers use techniques that require more computational power.
No matter how good the optimized code is (using specific data structures, algorithms etc) these problems will always take more time than easier ones due to their complexity.
Easy problems take less time than beginner problems and that is to be expected as well.
Programmers in this level have learned better coding practices and algorithms (from their mistakes in Beginner level problems) so their programs take less time.

Fastest language by execution time

Next up, the moment you've all been waiting for! The fastest programming language!
This chart shows the time taken for program solutions for each language on average. C and ADA get the number one place with a very small difference on ADA's time over C.
The worst languages are F#, LISP Sbcl and SCM guile. C and ADA might be the fastest, but most competitive programmers use C++ in its newest version
(currently C++ 20) for their solutions.
