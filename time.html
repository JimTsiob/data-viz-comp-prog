Time Criteria

Time Criteria is a category of visualizations that has to do -as the title says- with the criterion of time, which is very significant when competing at this sport. Execution or running time of a solution is a strong indicator of how smart an algorithm may be and how much the programmer who implemented it knows inside-out the coding language but also how to use data structures in general. The faster the solution, the better but a programmer must keep in mind not to exceed the space limit, a criterion we'll see in the next page.

Average execution time per difficulty level

Time is of the essence! This chart shows how much time a solution takes (in milliseconds) for every level of difficulty.

Harder difficulty problems require more time to complete because programmers use techniques that require more computational power. No matter how good the optimized code is (using specific data structures, algorithms etc) these problems will always take more time than easier ones due to their complexity. Easy problems take less time than beginner problems and that is to be expected as well. Programmers in this level have learned better coding practices and algorithms (from their mistakes in Beginner level problems) so their programs are running in less time.

Fastest language by execution time

Next up, we are investigating the fastest programming language. This bar chart visualizes the execution time of problems' solutions for each language on average.

C and ADA get the number one place with a very small difference on ADA's time over C. The worst languages are F#, LISP Sbcl and SCM guile. C and ADA might be the fastest, but most competitive programmers use C++ in its newest version (currently C++ 20) for their solutions.
