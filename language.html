If programming is war,
coding languages are the weapons

This page contains diagrams related to the programming languages that users picked to solve the problems. Generally, four are mostly picked among others which makes sense due to their speed and their pre-existing popularity.

Solutions per language

To get a quick overview, we start by investigating which languages are mostly picked among programmers to solve these problems.

As one can see in the donut chart, C++ is used in most solutions with 51.1% and then goes C with 29.1%. Both languages are known for their good time performance. Next come JAVA and PYTHON. We understand that competitive programmers code with languages that are faster even though they have more verbosity instead of coding with syntactically simpler ones but being slower (like PYTHON).

Lines Of Code per language

As the title says, this bar chart visualizes how many Lines of Code (LoC) on average are written with a language.

The 3 languages with the most lines are C++ followed by JAVA and C. These is because of their verbose nature to express things when writing code. Opposingly, TCL, HASK and PERL have the fewer LoC. Of course, this metric can be partly influenced by the personal style of each developer.

Changed Preferences

For the final visualization of this section, we found interesting to see how many users moved from their initial coding language to another. The Chord chart was the best fit for this purpose.

In general, most programmers that started with a language, kept coding in it. C++ was the language to welcome the most newcomers from other languages while keeping its majority of users (almost 22,000). "Population exchanges" between the rest of the languages ranged from 0 to 800.
